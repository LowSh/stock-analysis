# 股票分析日志更新检查清单

## 📋 更新前检查

### 1. 时间检查
- [ ] 确认当前时间（执行 `date "+%Y-%m-%d %H:%M:%S"`）
- [ ] 确认要更新的日期（今天/昨天）
- [ ] 确认上次更新时间（读取文件中的"最后更新"字段）

### 2. 数据源检查
- [ ] 确认新闻来源的时间戳是否合理
- [ ] 确认新闻不是未来时间（不应超过当前时间）
- [ ] 确认新闻时间顺序（从早到晚）

## 📝 更新时检查

### 3. 内容时间节点检查

#### 3.1 事件时间
- [ ] 所有事件时间 ≤ 当前时间
- [ ] 同一天的事件按时间从早到晚排列
- [ ] 跨天事件标注清楚（如"3/4 20:27"、"3/5 10:41"）

#### 3.2 板块热度时间
- [ ] 板块标注的时间范围正确（如"3/2-3/5持续强势"）
- [ ] "今日表现"标注的日期正确（如"3/5早盘"）
- [ ] "历史表现"与"今日表现"区分清楚

#### 3.3 市场数据时间
- [ ] 市场数据时间标注清楚（如"3/5 11:10更新"、"3/5收盘数据"）
- [ ] 盘中数据标注"午盘/早盘"
- [ ] 收盘数据标注"收盘"

### 4. 文件元数据检查
- [ ] 文件头部的"最后更新"时间 = 实际更新时间
- [ ] 文件底部的"最后更新"时间 = 文件头部时间
- [ ] 时间格式统一：`YYYY-MM-DD HH:MM`

## ✅ 更新后检查

### 5. 逻辑一致性检查
- [ ] 文件中提到的"今天"是否确实是更新日期
- [ ] "昨日回顾"是否确实是前一天
- [ ] "晚间"事件是否在18:00之后
- [ ] "早盘"事件是否在09:30-11:30之间
- [ ] "午盘"事件是否在13:00-15:00之间

### 6. 时间冲突检查
- [ ] 新添加的事件时间 ≥ 文件中已有的事件时间（同一天内）
- [ ] "最新进展"部分的事件确实是最新的
- [ ] 没有时间倒流（如先出现17:00的事件，后出现16:00的事件）

## 🔍 示例检查

### ❌ 错误示例
```
当前时间：2026-03-05 17:41

错误1：文件中写"3/5 18:30 某事件"（未来时间）
错误2：先写"17:07 事件A"，后写"16:47 事件B"（时间倒流）
错误3：文件头写"最后更新：17:15"，但添加了17:30的新闻（时间不匹配）
错误4：板块热度写"3/2-3/3"，但今天是3/5（时间过期）
```

### ✅ 正确示例
```
当前时间：2026-03-05 17:41

正确1：所有事件时间 ≤ 17:41
正确2：事件按时间排序：16:47 → 17:02 → 17:03 → 17:07 → 17:10
正确3：文件头"最后更新：17:41"（或最新事件时间）
正确4：板块热度更新为"3/2-3/5持续强势"
```

## 📊 检查命令

### 获取当前时间
```bash
date "+%Y-%m-%d %H:%M:%S"
```

### 检查文件中的时间标注
```bash
grep -n "最后更新\|_最后更新" 文件名.md
```

### 检查事件时间（提取所有时间戳）
```bash
grep -oE "[0-9]{1,2}:[0-9]{2}" 文件名.md | sort -u
```

## 🎯 快速检查脚本

```bash
#!/bin/bash
# 股票分析日志更新检查脚本

echo "=== 股票分析日志更新检查 ==="
echo ""

# 1. 获取当前时间
CURRENT_TIME=$(date "+%Y-%m-%d %H:%M:%S")
echo "✓ 当前时间：$CURRENT_TIME"
echo ""

# 2. 检查文件更新时间
echo "✓ 文件最后更新时间："
grep -n "_最后更新\|^最后更新" ~/Desktop/claw实验/股票分析日志.md
echo ""

# 3. 检查最新事件时间
echo "✓ 文件中最新事件时间（前5个）："
grep -E "^\s*-\s*\*\*[0-9]{1,2}:[0-9]{2}\*\*" ~/Desktop/claw实验/股票分析日志.md | head -5
echo ""

echo "=== 检查完成 ==="
```
